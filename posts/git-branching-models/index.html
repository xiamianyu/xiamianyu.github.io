Git分支模型

夏眠鱼
2024/06/24
vcs

本分支管理模型以主干分支为核心，围绕功能开发、代码优化、版本发布和生产应急修复四大场景，设计了五类分工明确的分支，既保障了生产环境的稳定性，又适配了日常开发的灵活性。整体遵循 “主干永久、分支临时、合入即删” 的核心原则，形成一套闭环的 “开发 -> 测试 -> 发布 -> 修复” 流程。

分支总览

分支类型 核心用途 生命周期
主干分支 生产环境 永久
特性分支 功能开发 临时
优化分支 代码优化 临时
发布分支 版本集成/测试/发布 临时
修复分支 生产应急修复 临时

主干分支

  • 命名规范：master
  • 核心用途：生产环境
  • 生命周期：永久存在
gitGraph
  commit
   commit tag: "v1.0.0"
   commit
   commit tag: "v1.1.0"

特性分支

  • 命名规范：feature/<描述>
  • 核心用途：功能开发
  • 生命周期：
    1. 基于主干创建
    2. 开发完成后合入发布分支
    3. 当发布分支合入主干后删除特性分支
gitGraph
   commit
   commit tag: "v1.0.0"
   branch feature/aaa
   commit
   checkout main
   branch feature/bbb
   commit

优化分支

作为特性分支的补充，只做代码优化

  • 命名规范：optimize/<描述>
  • 核心用途：代码重构、性能优化、架构调整、规范对齐等代码优化
  • 生命周期：与特性分支一致
gitGraph
   commit
   commit tag: "v1.0.0"
   branch optimize/aaa
   commit
   checkout main
   branch optimize/bbb
   commit

发布分支

  • 命名规范：release/<版本>
  • 核心用途：集成/测试/发布
  • 生命周期：
    1. 基于主干创建
    2. 合并需要发布的特性分支
    3. 验收通过后，打版本号，合入主干，删除发布分支
gitGraph
   commit
   commit tag: "v1.0.0"
   branch feature/aaa
   commit
   commit
   checkout main
   branch feature/bbb
   commit
   commit
   checkout main
   branch release/1.1.0
   merge feature/aaa
   merge feature/bbb
   commit tag: "v1.1.0"
   checkout main
   merge release/1.1.0

修复分支

  • 命名：hotfix/<描述>
  • 作用：生产环境异常修复
  • 生命周期：
    1. 基于主干创建
    2. 验收通过后，打版本号，合入主干，删除修复分支
gitGraph
   commit
   commit tag: "1.0"
   branch hotfix/aaa
   commit tag: "v1.0.1"
   checkout main
   merge hotfix/aaa
