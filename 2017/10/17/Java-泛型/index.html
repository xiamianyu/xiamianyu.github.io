为什么使用泛型

简而言之，在定义类、接口和方法时，泛型能让类或接口变成参数，这样的参数又叫类型参数（Type Parameter）。就像方法声明中的形参一样，类型参数提供了代码复用的通道，不同的是形参接收的是值，类型参数接收的是类型。

使用泛型的好处：

  • 编译时强类型检查
  • 消除类型转化
  • 实现通用算法

泛型

泛型（Generic Type）是带类型参数的或接口。下面我们通过 Box 类来论述什么是泛型。

泛型版本的 Box

可以看到，所有的 Object 都被替换成 T。我们称 T 为类型参数（Type Parameter）或类型变量（Type Variable），它可以是任何的非基本类型：类、接口、数组、甚至是类型参数。

根据上面的例子，我们可以得出泛型类的定义格式：
class name<T1, T2, ..., Tn> {/* ... */}

用尖括号（<>）分割的类型参数，遵循类名。泛型接口的定义和泛型类相似。

泛型参数的命名惯例

按照惯例，泛型参数的命名是单个大写字母，这样可以很好地区分类型变量和普通类或接口。

最常用的类型变量参数如下：

  • E - Element
  • K - Key
  • V - Value
  • T - Type
  • N - Number
  • S,U,V etc. - 2nd, 3rd, 4th types

Type Paramemter 和 Type Argument

虽然 Type Parameter 和 Type Argument 在中文上都是叫类型参数，但两者是有区别的。在编写代码时，为了创建参数化类型，可以提供类型参数（Type Argument）。因此，Box<T> 中的 T 是一个 Type Paramemter，而 Box<Integer> 的 Integer 是一个 Type Argument。
