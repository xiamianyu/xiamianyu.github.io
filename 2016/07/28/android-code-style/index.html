Android命名规范

团队开发中，统一命名规范，可以减少组员之间交接和沟通的成本，代码规范没有严格的好与坏，只要做到见名知义，方便梳理即可。

Java文件

采用驼峰命名法

Activity

以Activity作为后缀，如：OrderActivity

Fragment

以Fragment作为后缀，如：NewsFragment

Adapter

以Adapter作为后缀，如：NewsAdapter

常量

全部大写，单词之间用下划线分开，如：public final String CACHE_PATH = “xxx”

静态变量

首字母小写，加前缀s，如：public static String sName

成员变量

首字母小写，加前缀m，如：private int mAge

局部变量

首字母小写，不用加前缀，如：int age

控件变量

Android的控件变量，采用“逻辑功能+控件缩写”的形式，如：登陆按钮为loginBtn。我有个强迫症，变量声明也要排版一下，static变量放在最前面，控件变量放中间，普通成员变量放在后面，示例如下：

layout

格式：

  • 界面类型_模块.xml
  • 界面类型_模块_描述.xml

例如：

  • activity_login.xml
  • fragment_login.xml
  • dialog_login.xml

layout-item

格式：

  • item_控件类型_描述
  • item_控件类型_模块_描述

例如：

  • item_listview_order.xml
  • item_gridview_main_photo.xml

layout-include/merge

格式：

  • 包含类型_模块_描述
  • 包含类型_控件类型_描述

例如：

  • merge_detail_footer.xml
  • include_refreshview_header.xml

layout-id

格式：模块名_控件类型_描述

例如：activity_category.xml文件下显示姓名的TextView，其命名为android:id="@+id/category_tv_name"

drawable-xml

格式：

  • common_描述：多个模块共用
  • 模块_描述：单个某块共用

例如：

  • common_delete.xml (共用)
  • theme_add.xml （添加主题按钮）

drawable-png

格式：

  • 类型_common_描述：多个模块共用
  • 类型_模块_描述：单个模块共用
  • 类型_模块_描述_状态：配合动态风格使用

例如：

  • ic_common_delete.png (删除按钮，多个模块共用)
  • ic_accont_head.png（账户模块下的头像）
  • ic_theme_add_nomal.png (添加主题按钮的正常状态)
  • ic_theme_add_pressed.png (添加主题按钮的被按下状态)

style.xml

Android中，styles这个文件名称并没有作用，起作用的是在文件里xml的 <style>标签。因此你可以有多个style文件，如：

  • styles.xml
  • style_home.xml
  • styles_forms.xml
