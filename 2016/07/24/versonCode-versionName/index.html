versonCode & versionName

Posted on 2016-07-24   |   In Android

学艺不精

Android 开发中，我们对 versionCode 和 versionName 都不陌生，可能有些人对它们的理解还停留在 versionCode 对开发可见（对用户不可见），int 类型，versionName 对用户可见，String 类型。每次发布版本时都要修改 versionCode 和 versionName。最近因为版本控制命名问题造成的困扰，我才去 Android 官网查阅关于它们的介绍。

官网定义

versionCode — An integer used as an internal version number. This number is used only to determine whether one version is more recent than another, with higher numbers indicating more recent versions. This is not the version number shown to users; that number is set by the versionName setting, below.

The value is an integer so that other apps can programmatically evaluate it, for example to check an upgrade or downgrade relationship. You can set the value to any integer you want, however you should make sure that each successive release of your app uses a greater value. The system does not enforce this behavior, but increasing the value with successive releases is normative.

Typically, you would release the first version of your app with versionCode set to 1, then monotonically increase the value with each release, regardless whether the release constitutes a major or minor release. This means that the versionCode value does not necessarily have a strong resemblance to the app release version that is visible to the user (see versionName, below). Apps and publishing services should not display this version value to users.

Warning: The greatest possible value for versionCode is MAXINT (2147483647). However, if you upload an app with this value, your app can’t ever be updated.

versionName — A string used as the version number shown to users. This setting can be specified as a raw string or as a reference to a string resource.
The value is a string so that you can describe the app version as a <major>.<minor>.<point> string, or as any other type of absolute or relative version identifier. The versionName has no purpose other than to be displayed to users.

理解消化

可以看到，官网对 versionCode 和 versionName 做了很详细的说明，用程序员能理解的总结如下：

versionCode

类型：MAXINT(2e31 = 2147483647)
作用：标识 App 内部版本，对开发可见（对用户不可见），建议每次发布 release 版本时累加1
注意：长度不能超过 MAXINT(2e31 = 2147483647)，否则 App 将无法再升级。

特别的，Android 中，覆盖安装时，不允许对 versionCode 做降级处理（按常理，没必要降级），如：上一版 versionCode=”2”，新版 versionCode 不能小于2，否则覆盖安装失败。AS 上做了降级测试，出现以下提示：

versionName

类型：String
格式：<major>.<minor>.<point>（<主版本号>.<子版本号>.<修订版本号>）
作用：标识 App 版本，对用户可见
