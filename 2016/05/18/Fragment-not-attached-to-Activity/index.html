Fragment not attached to Activity

Posted on 2016-05-18   |   In Android Bug   |     |   visitors

异常日志

原因分析

从异常日志中可以定位到 Fragment 的 getResources() ，OK，看下 getResources() 的源码。

从源码得知，当 mHost 为 null 时, 就会抛出该异常。mHost 是啥东西，根据 mHost.getContext().getResources() 我们可以推断 mHost.getContext() 相当于 getActivity()，即异常的原因是 getActivity() 为 null。

造成 getActivity() 为 null 的场景可能有很多种，以 ViewPager + Fragment + FragmentStatePagerAdapter 组合为例，AFragment 中异步请求数据，请求成功我们就调用 getString() ，在请求没有成功之前（网络很差），我们切换到BFragment，刚好这时，ViewPager 回收 AFragment （即 AFragmemnt 的 onDetach() 被执行），过了一会，AFragment 的请求成功并调用其 getString()，这时 getActivity() 为 null，就会报 Fragment not attached to Activity 异常。

解决方法

1. 添加 isAdd() 判断

isAdd() 是Fragment的方法，为 true 时，表示当前 Fragment 被添加到 Activity，即 getActivity() 不为 null。

2. 断开异步请求

当 Fragment 的 onDestroy() 被回调时，断开异步请求。为防止断开失败，还是添加 isAdd() 做双重保险。
