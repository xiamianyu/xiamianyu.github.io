初尝面试官

发表于 2016-05-15   |   分类于 人力资源   |     |   阅读次数

前言

第一次当面试官，激动且有点不知所措，以前都是别人面试我，现在轮到我面试别人，我应该怎么面试呢（想到大四去教育实习的经历）？在团队管理过程中，多少有了解面试方面的知识，但都是理论性的，自己没实战过，心里还是空落落的。这两天的面试，让我对它有了全新的认识，也深刻意识到其重要性。

筛选简历

这两天面试了五六个人，没有一个合格，看了简历之后，最大的感觉是，为什么他们能得到面试的机会。简历排版这么差、内容描述没重点、罗里吧嗦、简历信息不真实等，好浪费时间呐，如果你还不懂怎么写出合格的简历，请参考：

80% 以上简历都是不合格的
推荐两个技术简历模板

回归正题，「为什么他们能得到面试的机会？」，后来我才知道，HR 看到有简历投递过来就预约面试，这显然是有问题的。能得到面试机会的简历应该是被认为有能力胜任这个职位的人，因此简历务必经过开发部的审核，确定有意向再预约，这样可以大大提高面试的质量，找到合适的人。

笔试题目

IT 的面试，大多都有笔试，用于考察应聘者的硬技能，笔试的题目怎么出不可忽视。从这两天的面试效果来看，之前的笔试题目，太过简单，全都是基于 Android 最基础的知识点。总的来说，无法通过笔试来初次判断应聘者的功底，我认为是失败的。后来我临时补充了一些，加了 Java 基础和 Android 其他方面的 知识点，效果好了一些。笔试内容可以是 Java + Android ，从浅到深，有区分度，更好地考察应聘者的功底。

软技能

软技能指的是沟通能力、抗压能力、性格等，沟通能力在面试交流时，可以察觉到一些，性格、抗压能力，短短的一个面试还是挺难考察的。

以上是我作为一位小白面试官的经历，还是要多多学习。下面这篇文章是在知乎上看到的，有很大参考价值，值得好好学习。
